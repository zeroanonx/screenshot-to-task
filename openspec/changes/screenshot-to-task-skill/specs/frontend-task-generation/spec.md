## 新增需求

### 需求:分层任务拆解

Skill 必须将截图解析结果拆解为固定分层的前端开发任务清单。

#### 场景:标准分层结构

- **当** Skill 完成截图解析
- **那么** 产出文档必须包含以下章节（可按需省略无内容的章节，但不得打乱顺序）：需求摘要、路由与菜单、API 与类型、常量与枚举、页面、组件、交互与状态流转、校验与错误处理、权限、部署与联调顺序、验证清单、范围外

#### 场景:任务条目格式

- **当** Skill 生成单条任务
- **那么** 必须使用 `- [ ]` checkbox 格式，包含可执行描述，并在有来源时标注截图编号或 PRD 章节

### 需求:交互与状态建模

Skill 必须对涉及状态流转的功能建立显式模型。

#### 场景:状态机提取

- **当** 截图或 PRD 定义了业务状态（如待开票、待收款、已收款）
- **那么** Skill 必须在任务文档中描述完整状态机和各状态下的可用操作

#### 场景:操作矩阵

- **当** 不同状态下展示不同操作按钮
- **那么** Skill 必须生成操作矩阵任务项，明确每个状态下可见的按钮及其触发行为

### 需求:可选代码库映射

Skill 在用户项目包含前端源码时，必须将任务映射到具体代码位置。

#### 场景:有代码库项目

- **当** 用户项目存在 `package.json` 和 `src/` 目录
- **那么** Skill 必须扫描项目结构，将任务细化到具体文件路径，并在可能时提供代码片段和行号参考

#### 场景:无代码库或纯 PRD 阶段

- **当** 用户项目无可映射的前端源码
- **那么** Skill 必须生成职责级任务，文件路径使用 `{待映射}` 占位，不得编造不存在的路径

### 需求:业务规则完整性

Skill 必须保留 PRD 中的关键业务规则，不得遗漏影响前端实现的规则。

#### 场景:计算公式

- **当** PRD 包含金额计算公式或税率规则
- **那么** Skill 必须在任务文档中原文保留公式和精度要求（如保留两位小数）

#### 场景:校验与提示文案

- **当** PRD 定义了前端校验或 Toast 提示
- **那么** Skill 必须使用 PRD 原文文案写入任务，禁止自行改写
