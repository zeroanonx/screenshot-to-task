## 新增需求

### 需求:截图分类与内容提取

Skill 必须对用户提供的每张截图进行分类，并提取前端实现所需的关键信息。

#### 场景:UI 稿页面识别

- **当** 用户提供包含页面布局的 UI 截图
- **那么** Skill 必须识别页面标题、菜单路径、筛选项、表格列、按钮、状态标签、分页组件和抽屉/弹窗结构

#### 场景:PRD 交互说明识别

- **当** 用户提供包含文字说明的 PRD 截图
- **那么** Skill 必须提取业务规则、状态流转、校验逻辑、Toast 提示文案、计算公式和权限要求

#### 场景:多图关联

- **当** 用户提供多张相关截图
- **那么** Skill 必须识别页面跳转关系、流程编号（如 01/02）和图与图之间的层级关系

### 需求:信息源优先级

Skill 在解析冲突信息时必须遵守固定优先级。

#### 场景:PRD 与 UI 冲突

- **当** PRD 文字说明与 UI 视觉呈现存在差异
- **那么** Skill 必须以 PRD 文字为准，并将差异记录到产出文档的「待确认项」章节

#### 场景:仅 UI 无 PRD

- **当** 截图仅包含 UI 无文字说明
- **那么** Skill 必须基于 UI 推断交互逻辑，并标注推断来源为「UI 视觉推断」

### 需求:多端多功能识别

Skill 必须能够区分不同端和独立功能模块。

#### 场景:多端截图混合

- **当** 截图包含不同端（如小程序与管理后台）的内容
- **那么** Skill 必须按端拆分，并为每个端生成独立的任务名称

#### 场景:同端多功能

- **当** 截图包含同一端下的多个独立功能
- **那么** Skill 必须为每个功能生成独立的任务文档，除非用户明确要求合并
