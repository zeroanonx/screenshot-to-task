## 新增需求

### 需求:独立文件产出

Skill 必须将任务清单写入用户项目根目录的独立 Markdown 文件。

#### 场景:标准产出路径

- **当** Skill 完成任务生成
- **那么** 必须将文件写入 `{用户项目根目录}/task/{中文任务名称}.md`，并自动创建 `task/` 目录（若不存在）

#### 场景:不写入 OpenSpec

- **当** Skill 完成产出
- **那么** 禁止默认写入 `openspec/changes/` 目录；OpenSpec 对接由用户自行决定

### 需求:中文任务名称自动生成

Skill 必须根据截图内容自动生成中文任务名称作为文件名。

#### 场景:名称推断

- **当** Skill 分析截图内容
- **那么** 必须优先使用页面标题或菜单末级名称生成 2～12 个汉字的中文名称

#### 场景:非法字符处理

- **当** 推断的名称包含文件系统非法字符（`/` `\` `:` `*` `?` `"` `<` `>` `|`）
- **那么** 必须移除或替换非法字符后再用作文件名

#### 场景:重名处理

- **当** `task/` 目录下已存在同名文件
- **那么** 必须在文件名后追加序号（如 `补贴收款管理-2.md`）

### 需求:生成后名称确认提醒

Skill 必须在产出完成后提醒用户确认或修改任务名称。

#### 场景:固定提醒话术

- **当** 任务文件写入完成
- **那么** Skill 必须告知用户：文件路径、自动推断的任务名称，并询问是否需要修改名称；若用户要求修改，必须重命名文件并同步更新文档标题

### 需求:文档标题与元信息

产出文件必须包含标准化的文档头部。

#### 场景:文档结构

- **当** 写入 `task/{中文任务名称}.md`
- **那么** 文档第一行必须为 `# Tasks: {任务名称}（前端）`，并包含来源截图数量、文件路径等元信息块

### 需求:输入通用性

Skill 不得将特定业务域绑定为固定任务。

#### 场景:示例截图

- **当** 仓库或文档中包含补贴、收款等示例截图
- **那么** 这些截图仅作为 skill 输入格式示例，不得硬编码为 skill 的默认输出内容
