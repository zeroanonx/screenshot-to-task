# 任务输出模板

> 与 [SKILL.md](../SKILL.md) 配套。生成 `task/{中文任务名称}.md` 时严格遵循本模板。

## 文档头部

```markdown
# Tasks: {任务名称}（前端）

> 来源：{N} 张截图
> 业务线：{业务线名称}
> 端：{小程序 / H5 / 管理后台}
> 文件：task/{任务名称}.md
> 截图目录：task/screenshot/{任务名称}/

## 截图索引

| 序号 | 文件 | 说明 |
|------|------|------|
| 01 | [01-列表页](screenshot/{任务名称}/01-列表页.png) | 列表页 UI |
| 02 | [02-PRD交互说明](screenshot/{任务名称}/02-PRD交互说明.png) | 交互 PRD |
```

## 固定章节顺序

```markdown
## 0. 整体任务预览

## 1. 路由与菜单

## 2. API 与类型

## 3. 常量与枚举

## 4. 页面 — {PageName}

## 5. 组件 — {ComponentName}

## 6. 交互与状态流转

## 7. 校验与错误处理

## 8. 权限

## 9. 验证清单

## 10. 范围外

## 11. 待补充项
```

**`## 0. 整体任务预览` 必填且为第一个章节，不可省略。**
**`## 11. 待补充项` 必填且为最后一个章节，不可省略。**
无内容的其他章节可省略，但已有章节不得打乱顺序。

**不包含**：部署与联调顺序、后端实现逻辑。

---

## §0 整体任务预览（必填模板）

产出文档的第一个章节，用一段话 + 结构化列表让读者 30 秒内掌握全貌：

```markdown
## 0. 整体任务预览

### 业务线

{业务线名称}（如：会员营销 / 补贴管理 / 商户端运营）

### 做什么

{1～3 句话：本功能的产品目标，用户要完成什么业务动作}

### 怎么做

{前端实现策略概述：涉及几个页面、核心交互模式、主要改造点}

### 页面全景

| 页面 | 类型 | 职责 |
|------|------|------|
| {页面A} | 新建 | {一句话职责} |
| {页面B} | 改造 | {一句话职责} |
| {弹窗C} | 新建组件 | {一句话职责} |

### 核心用户路径

```text
{入口菜单}
  → {页面A}：{用户在此做什么}
    → 点击「{按钮}」→ {结果：跳转/弹窗/提交}
      → {页面B/弹窗}：{用户在此做什么}
        → 点击「{按钮}」→ {终点}
```

或使用 mermaid：

​```mermaid
flowchart LR
  A[入口菜单] --> B[列表页]
  B -->|点击「新增」| C[表单页]
  C -->|提交成功| B
​```

### 点击操作一览

| # | 所在页面 | 点击 | 结果 |
|---|---------|------|------|
| 1 | 列表页 | 「查询」 | 刷新表格数据 |
| 2 | 列表页 | 「查看详情」 | 打开详情抽屉 |
| 3 | 详情抽屉 | 「通过」 | 二次确认弹窗 → 提交 → 刷新列表 |

### 待确认项

- {读图阶段未完全确认、但已记录的问题，无则写「无」}
```

## 单条任务格式

```markdown
- [ ] {章节号}.{序号} {可执行描述}
  - 来源：[01-列表页](screenshot/{任务名称}/01-列表页.png)
```

### 有代码库映射

```markdown
- [ ] 4.1 新建 `src/views/Example/List/index.vue`（**新建页面**）
  - 来源：[01-列表页](screenshot/{任务名称}/01-列表页.png)
```

### 无代码库映射

```markdown
- [ ] 4.1 创建列表页 `{待映射}`（**新建页面**）
  - 来源：[01-列表页](screenshot/{任务名称}/01-列表页.png)
```

## 章节内容指南

### 1. 路由与菜单

- 路由 path、name
- 菜单权限点
- 标注哪些是**新增路由**

### 2. API 与类型

**仅写前端消费层**：

- 需调用的接口 path、method
- 前端 TS 类型/interface
- 筛选项与请求参数的映射

**不写**：后端实现、数据库设计、定时任务逻辑

### 4. 页面

每个页面注明 **新建** 或 **改造**，写清：

- 页面结构（筛选区 / 表格 / 表单）
- 从哪来、到哪去（入口和出口）

### 6. 交互与状态流转

- 完整点击操作表（可复用 §0 并展开细节）
- 状态机 + 操作矩阵（如有）

### 7. 校验与错误处理

- 前端校验规则
- Toast 文案（PRD 原文）
- 接口错误的前端展示

### 10. 范围外

**后端逻辑、部署联调等一律放这里**，不作为任务项：

```markdown
## 10. 范围外

- 后端接口实现
- 定时任务 / 数据生成逻辑
- 第三方系统对接（财务中台等）
```

### 11. 待补充项

```markdown
## 11. 待补充项

> 以下由用户补充，AI 生成时留空待填。

- [ ] 11.1
- [ ] 11.2
```

生成后提醒：「请检查并填写第 11 章「待补充项」。」
