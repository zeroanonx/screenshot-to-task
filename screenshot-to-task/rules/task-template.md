# 任务输出模板

> 与 [SKILL.md](../SKILL.md) 配套。生成 `task/{中文任务名称}.md` 时严格遵循本模板。

## 文档头部

```markdown
# Tasks: {任务名称}（前端）

> 来源：{N} 张截图
> 业务线：{业务线名称}
> 端：{小程序 / H5 / 管理后台}
> 文件：task/{任务名称}.md
> 截图目录：task/screenshot/{任务名称}/

## 截图索引

| 序号 | 文件 | 说明 |
|------|------|------|
| 01 | [01-列表页](screenshot/{任务名称}/01-列表页.png) | 列表页 UI |
| 02 | [02-PRD交互说明](screenshot/{任务名称}/02-PRD交互说明.png) | 交互 PRD |
```

## 固定章节顺序

```markdown
## 0. 整体任务预览

## 1. 路由与菜单

## 2. 常量与枚举

## 3. 页面 — {PageName}

## 4. 组件 — {ComponentName}

## 5. 交互与状态流转

## 6. 校验与错误处理

## 7. 权限

## 8. 范围外

## 9. 待补充项
```

**`## 0. 整体任务预览` 必填、第一个章节，且须尽可能详细。**
**`## 9. 待补充项` 必填、最后一个章节，不可省略。**

**不包含的章节**：API 与类型、验证清单、部署与联调顺序、后端实现逻辑。

---

## §0 整体任务预览（必填，尽可能详细）

§0 是整份文档最重要的章节，读者应**无需翻截图和后续章节**即可完整理解需求。禁止敷衍，每项都要写足。

```markdown
## 0. 整体任务预览

### 业务线

{业务线名称} — {一句话说明该业务线在产品中的定位}

### 做什么

{产品背景：为什么做这个功能}

{产品目标：用户（角色）要完成什么业务动作}

{预期结果：操作完成后业务状态如何变化}

### 怎么做

{前端整体方案：涉及几个页面/组件、核心交互模式}

{技术要点：如条件渲染、多步骤表单、抽屉嵌套等}

{与已有功能的关系：改造还是全新}

### 页面全景

| 页面/组件 | 类型 | 路由/挂载 | 职责 | 关键元素 |
|-----------|------|-----------|------|---------|
| {页面A} | 新建 | {路由或父页面} | {详细职责} | {筛选项、表格列、按钮等} |
| {页面B} | 改造 | {已有路由} | {改什么} | {新增/变更的元素} |
| {组件C} | 新建组件 | {挂载在哪个页面} | {职责} | {触发方式、内容结构} |

### 字段与展示

| 页面/组件 | 字段/列名 | 类型 | 说明 | 必填 | 来源 |
|-----------|----------|------|------|------|------|
| 列表页 | 会员姓名 | 文本 | 表格列 | — | 图01 |

### 核心用户路径

```text

  → {页面A}（{用户在此做什么，看到什么}）

## §1 路由与菜单（须先询问用户）

**生成 §1 之前，必须先向用户确认路由方案，不得自行编造路由 path。**

### 询问流程

**第一步：是否新建路由**

```text
该功能是否需要新建路由？
- A. 新建路由
- B. 不新建，挂载到已有路由/页面
```

**选 A → 新建路由：严格询问用户**

```text
请提供新路由信息：
1. 路由 path（如 /member/points-exchange）
2. 路由 name（如 PointsExchange）
3. 菜单挂载位置（如：会员营销 → 积分兑换）
```

用户未提供前**不得写入任务文档**，不得自行推断 path/name。

**选 B → 不新建：询问已有路径**

```text
请提供已有路由信息：
1. 源路由 path
2. 源路由 name
3. 页面文件路径（如已知）
4. 本次是新增子路由、子页面还是在已有页面内改造？
```

### §1 任务写法

根据用户确认结果，写**项目中具体怎么改**：

```markdown
## 1. 路由与菜单

- [ ] 1.1 **新建路由** `path: '/member/points-exchange'`，`name: 'PointsExchange'`
  - 菜单：会员营销 → 积分兑换
  - 用户确认：{用户提供的原始信息}
  - 来源：[01-列表页](screenshot/{任务名}/01-列表页.png)

- [ ] 1.2 在 `src/router/modules/member.ts` 追加路由配置（**按项目实际路由文件**）
  ```ts
  { route config 示例 }
  ```
```

或不新建时：

```markdown
## 1. 路由与菜单

- [ ] 1.1 **不新建路由**，在已有路由 `path: '/member/list'`（`name: 'MemberList'`）上改造
  - 页面文件：`src/views/Member/List/index.vue`
  - 用户确认：{用户提供的原始信息}
```

## 章节内容指南

### 2. 常量与枚举

- 状态枚举 + 标签色/文案
- 表格列配置
- 操作按钮矩阵

### 3. 页面

每个页面注明 **新建** 或 **改造**，写清结构、字段、从哪来到哪去。

### 5. 交互与状态流转

- 展开 §0 点击操作表的实现细节
- 状态机流转

### 6. 校验与错误处理

- 前端校验、Toast 原文、错误展示

### 8. 范围外

后端逻辑、API 实现、部署联调等，不作为任务项。

### 9. 待补充项

```markdown
## 9. 待补充项

> 以下由用户补充，AI 生成时留空待填。

- [ ] 9.1
- [ ] 9.2
```

生成后提醒：「请检查 §9 待补充项，可随时追加。」

### 落地前确认（Phase 5）

呈现理解摘要后，**必须 `AskQuestion` 选择框**：

```
选项 A. 理解准确，直接生成文档（Recommended）
选项 B. 理解有偏差，需要修正  → 用户输入偏差 → 修正 → 重新确认
选项 C. 需要补充内容          → 用户输入补充 → 纳入文档 → 重新确认
```

仅当用户选择 A 时执行落盘。选择 C 时必须先收集用户输入，不可跳过。
