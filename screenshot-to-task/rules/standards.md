# 任务生成质量标准

> 与 [SKILL.md](../SKILL.md) 配套。生成 `task/*.md` 时逐项检查。

## 一、必达项

| # | 规则 | 不满足时 |
|---|------|---------|
| 1 | 产出路径 `task/{中文任务名称}.md` | 阻塞交付 |
| 2 | 文档头部标注**业务线**和**端** | 阻塞交付 |
| 3 | **§0 整体任务预览**必填且为第一个章节 | 阻塞交付 |
| 4 | §0 须回答：做什么、怎么做、从哪到哪、新建页面、点击操作 | 阻塞交付 |
| 5 | 必须有验证清单、范围外、待补充项（§11 最后一章） | 阻塞交付 |
| 6 | 截图归档至 `task/screenshot/{任务名}/` 并引用 | 阻塞交付 |
| 7 | 读图不确定项须暂停等用户确认 | 阻塞交付 |
| 8 | 每条任务 `- [ ]` checkbox 格式 | 阻塞交付 |

## 二、产品意图理解

产出必须让读者无需翻截图即可理解：

- **业务线**是什么
- **产品目标**（做什么）
- **实现路径**（怎么做）
- **完整用户路径**（从哪到哪）
- **页面清单**（新建/改造）
- **点击操作表**（点什么出什么）

缺少任一项 → 任务不完整。

## 三、前端范围边界

**必须写**：

- 页面、组件、路由、菜单
- 前端交互、校验、状态展示
- API 消费层（调用什么接口、什么参数、什么 TS 类型）

**禁止写为任务项**：

- 后端接口实现
- 数据库设计、定时任务
- 部署与联调顺序
- 第三方系统内部逻辑

上述内容若截图涉及，仅记入 **§10 范围外**，不作为 `- [ ]` 任务。

## 四、可追溯性

- 标注来源：`来源：[01-xxx](screenshot/{任务名}/01-xxx.png)`
- 推断内容标注 `来源：UI 视觉推断`
- 不确定项：读图阶段暂停确认；已确认写入 §0 或对应章节

## 五、用户确认门禁

不清楚的交互、页面归属、业务线或产品意图 → **暂停** → 等用户确认 → 再继续。

## 六、原文保留

Toast、弹窗文案、公式、字段 label 使用 PRD 原文，禁止改写。

## 七、状态与交互

涉及状态时须有：状态枚举、流转描述、操作矩阵。§6 须有完整点击操作表。

## 八、禁止事项

- 禁止省略 §0 整体任务预览
- 禁止写入部署与联调顺序章节
- 禁止将后端逻辑写为前端任务项
- 禁止不保存截图就生成文档
- 禁止在读图有不确定项时跳过确认
- 禁止默认写入 `openspec/changes/`
