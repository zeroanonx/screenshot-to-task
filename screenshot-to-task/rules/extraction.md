# 截图内容提取 Checklist

> 与 [SKILL.md](../SKILL.md) 配套。Phase 0 归档截图，Phase 1 理解产品意图。

## Phase 0 — 截图归档

```text
task/screenshot/{中文任务名称}/{两位序号}-{简要描述}.{扩展名}
```

## 产品意图五问（Phase 1 必答）

读图时逐项落实，答案写入 §0 整体任务预览：

| # | 问题 | 提取要点 |
|---|------|---------|
| 1 | **做什么** | 产品目标、用户业务动作、解决什么问题 |
| 2 | **怎么做** | 前端页面/组件/交互如何实现 |
| 3 | **从哪里到哪里** | 入口菜单 → 中间页 → 终点的完整路径 |
| 4 | **新建什么页面** | 逐页标注：新建 / 改造 / 复用已有 |
| 5 | **点什么出什么** | 每个关键按钮 → 跳转/弹窗/抽屉/提交/状态变更 |

## 业务线识别

从以下来源推断业务线，写入文档头部和 §0：

- 侧边栏一级/二级菜单归属（如「补贴管理」「会员营销」）
- 产品 Logo / 平台名称（如「来逛呗·商圈营销」「万小」）
- PRD 标题或章节归属
- 用户补充说明

无法判断时列入待确认项，不得留空。

## 用户确认门禁（全程适用）

**任何阶段**有任何不理解、不确定、需猜测的内容 → **立即暂停** → 询问用户 → **用户确认后才继续**。

包括但不限于：交互跳转、按钮行为、字段含义、业务线、页面归属、PRD/UI 冲突、状态流转、路由、截图看不清。

暂停格式见 [SKILL.md](../SKILL.md)。禁止自行推断后静默继续。

## 落地前确认（Phase 5，落盘前必做）

写入 `task/*.md` **之前**：

1. 在对话中呈现理解摘要
2. **使用 `AskQuestion` 弹出选择框**（不可用纯文字代替）：

   | 选项 | 后续 |
   |------|------|
   | 理解准确，直接生成文档 | → Phase 6 落盘 |
   | 理解有偏差，需要修正 | → 请用户输入偏差 → 修正 → 重新弹框 |
   | 需要补充内容 | → **请用户输入补充** → 纳入文档 → 重新弹框 |

3. 循环直到用户选择「理解准确，直接生成文档」

**用户确认前禁止落盘。**

## 点击操作提取

对截图中每个可交互元素，逐条记录：

```text
{页面名} → 点击「{按钮/链接/行}」→ {结果}
```

结果类型：

| 类型 | 示例 |
|------|------|
| 页面跳转 | 列表页 → 详情页 |
| 打开弹窗 | 点击「申请开票」→ 开票须知弹窗 |
| 打开抽屉 | 点击「账单详情」→ 右侧抽屉 |
| 提交/请求 | 点击「确认」→ 调用接口 → Toast → 刷新 |
| 状态变更 | 点击「已开发票」→ 确认弹窗 → 状态变为待收款 |
| 无操作 | 纯展示字段 |

## 页面分类

| 类型 | 判断依据 | 任务标注 |
|------|---------|---------|
| 新建页面 | 截图出现新菜单/新路由，代码库无对应文件 | **新建** |
| 改造页面 | 已有页面新增区块/字段/按钮 | **改造** |
| 新建组件 | 弹窗/抽屉/浮层，无独立路由 | **新建组件** |
| 复用 | 与已有页面/组件一致 | **复用** |

## 截图分类

| 类型 | 重点提取 |
|------|---------|
| UI 稿 | 字段、按钮、表格列、状态标签 |
| PRD 文字 | 业务规则、校验、权限 |
| 流程标注 | 页面跳转、触发条件 |
| 弹窗/抽屉 | 触发按钮、内容、确认行为 |
| 表格 | 列名、操作列、分页 |
| 状态机 | 状态枚举、流转、操作矩阵 |

## 必提取字段

### 页面级

- [ ] 页面标题、菜单路径、面包屑
- [ ] 页面类型（新建/改造）
- [ ] 入口来源、出口去向

### 筛选 / 表格 / 表单 / 弹窗

- [ ] 筛选项名称、类型、约束
- [ ] 表格列、操作列按钮及显示条件
- [ ] 表单字段、必填、条件显示
- [ ] 弹窗触发按钮、确认/取消行为

## 前端 vs 后端边界

**写入任务文档**：页面、组件、路由（用户确认后）、交互、校验、常量、权限

**不写入任务，记入 §8 范围外**：后端实现、API 定义、定时任务、部署联调

**不出现在文档中的章节**：API 与类型、验证清单

## 路由确认（Phase 2.5）

生成 §1 前必须询问用户：

- 新建路由 → 严格获取 path、name、菜单位置
- 不新建 → 获取源 path、name、页面文件路径、改造方式

不得自行推断路由信息。

## 提取完成自检

- [ ] 五问均已回答，§0 素材充足
- [ ] 业务线已标注
- [ ] 字段表、点击操作表已完整
- [ ] 每个页面已标注新建/改造
- [ ] 用户路径从入口到终点已连通
- [ ] 全程无不理解项遗留（均已询问并获用户确认）
- [ ] 路由方案已向用户确认
- [ ] 落地前已呈现理解摘要并询问补充项
- [ ] 用户已明确确认后才落盘
