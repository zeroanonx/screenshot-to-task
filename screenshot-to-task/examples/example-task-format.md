## 0. 整体任务预览

### 业务线

会员营销

### 做什么

为运营人员提供积分兑换申请的审核能力：查看兑换列表、审核通过或拒绝会员的积分兑换礼品申请。

### 怎么做

新建积分兑换列表页（筛选 + 表格 + 分页），点击行操作打开详情抽屉，在抽屉内完成审核操作；涉及 1 个新页面、1 个新抽屉组件、1 个确认弹窗。

### 页面全景

| 页面/组件 | 类型 | 职责 |
|-----------|------|------|
| 积分兑换列表页 | 新建 | 筛选、展示兑换申请、操作入口 |
| 兑换详情抽屉 | 新建组件 | 展示申请详情、审核通过/拒绝 |
| 拒绝确认弹窗 | 新建组件 | 拒绝前二次确认 |

### 核心用户路径

```text
会员营销 → 积分兑换（入口菜单）
  → 列表页：筛选/浏览兑换申请
    → 点击「查看详情」→ 详情抽屉：查看会员/礼品/积分信息
      → 点击「通过」→ 提交 → Toast「操作成功」→ 关闭抽屉 → 刷新列表
      → 点击「拒绝」→ 确认弹窗「确认拒绝该兑换申请？」
        → 确认 → 提交 → Toast「操作成功」→ 关闭抽屉 → 刷新列表
```

### 点击操作一览

| # | 所在页面 | 点击 | 结果 |
|---|---------|------|------|
| 1 | 列表页 | 「查询」 | 按筛选条件刷新表格 |
| 2 | 列表页 | 「重置」 | 清空筛选并刷新 |
| 3 | 列表页 | 「查看详情」 | 打开详情抽屉 |
| 4 | 详情抽屉 | 「通过」 | 调用审核接口 → Toast → 关闭抽屉 → 刷新列表 |
| 5 | 详情抽屉 | 「拒绝」 | 打开确认弹窗 |
| 6 | 确认弹窗 | 「确认」 | 调用拒绝接口 → Toast → 关闭弹窗和抽屉 → 刷新列表 |
| 7 | 确认弹窗 | 「取消」 | 关闭弹窗，状态不变 |
