## 0. 整体任务预览

### 业务线

会员营销 — 商圈运营后台中负责会员权益与积分运营的功能模块。

### 做什么

**背景**：会员可使用积分兑换礼品，运营人员需审核兑换申请。

**目标**：为运营提供兑换申请列表查看与审核（通过/拒绝）能力。

**预期结果**：审核通过后会员积分扣减、兑换生效；拒绝后申请关闭。

### 怎么做

新建积分兑换列表页 + 详情抽屉 + 拒绝确认弹窗。列表支持筛选和分页，审核在抽屉内完成。涉及 1 个新页面、2 个新组件。

### 核心用户路径

```text
会员营销 → 积分兑换（侧边栏入口）
  → 列表页：默认加载全部申请，可按手机号/状态/时间筛选
    → 点击「查询」：按条件刷新表格
    → 点击「查看详情」：右侧滑出详情抽屉，传入当前行 id
      → 抽屉内查看会员/礼品/积分/申请时间
      → 点击「通过」：调用审核接口 → Toast「操作成功」→ 关闭抽屉 → 刷新列表
      → 点击「拒绝」：弹出确认弹窗「确认拒绝该兑换申请？」
        → 点击「确认」：调用拒绝接口 → Toast「操作成功」→ 关闭弹窗和抽屉 → 刷新列表
        → 点击「取消」：关闭弹窗，抽屉保持打开，状态不变
```
