分析用户提供的产品截图、UI 稿或 PRD 交互说明，生成结构化前端开发任务清单。

**必须先读取并严格遵循 `screenshot-to-task/SKILL.md` 及其 rules/ 规范文件。**

---

**输入**：`/screenshot-to-task` 后可选补充说明；截图通过对话附件提供。

**步骤**

1. **读取 Skill 规范**（SKILL.md + rules/ + examples/）

2. **检查输入** — 无截图则提示附上，不继续

3. **按阶段执行（Phase 6 之前禁止写入 task/*.md）**

   ```
   Phase 0    截图归档
   Phase 1    读图 + 五问
                └─ 有任何不理解 → 暂停询问 → 用户确认后才继续
   Phase 2    交互建模（仍有疑问 → 暂停询问）
   Phase 2.5  路由确认（询问用户新建 or 已有路径）
   Phase 3    对话中准备任务内容（不写文件）
   Phase 4    代码库映射（可选，不写文件）
   Phase 5    落地前确认 ← 不可跳过
                · 呈现理解摘要
                · 询问：理解是否准确？是否还有补充项？
                · 等待用户明确确认
                · 用户补充纳入文档
   Phase 6    落盘写入 task/*.md ← 最后一步
   ```

4. **产出约束**

   - §0 详尽必填；§1 路由来自用户确认
   - §9 待补充项最后一章
   - 禁止：API 与类型、验证清单章节
   - 用户当场补充 → 写入对应章节；后续再补 → §9 占位

5. **落盘后摘要**

   - 文件路径、任务名称、改名提醒

**护栏**

- 有任何不理解必须问，禁止猜测
- Phase 5 不可跳过，用户确认前禁止落盘
- 路由 path/name 禁止自行推断
- §0 禁止敷衍
